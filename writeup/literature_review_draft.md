# 2. Related Work

This section situates our Confidence-Informed Gated $k$NN framework within the contemporary trajectory of software engineering intelligence, focusing specifically on the shift toward reliability, retrieval, and probability bounding in 2024 and 2025.

## 2.1 Deep Learning for Defect and Vulnerability Prediction
The paradigm of automated software defect prediction and vulnerability detection has historically been dominated by fine-tuning pre-trained contextual transformers such as CodeBERT and GraphCodeBERT. More recently, the field has transitioned toward utilizing Large Language Models (LLMs) like CodeLlama, DeepSeek-Coder, and GPT-4. While LLM-based approaches achieve competitive F1 scores by framing detection as a generative or prompting task, they suffer from fundamental deployment bottlenecks: they are computationally prohibitive to run across million-line repositories, and more critically, they lack rigorous probability calibration. 

Recent literature (2024-2025) highlights that both masked language models and modern LLMs exhibit dangerous overconfidence when evaluated on complex vulnerability datasets like Devign or algorithmic classification datasets like CodeChef. When faced with out-of-distribution (OOD) code snippets, these models frequently produce high-confidence hallucinations. Consequently, the Software Engineering (SE) community is increasingly pivoting away from merely maximizing raw accuracy scores, focusing instead on quantifying predictive uncertainty and establishing "trustworthy" deployment pipelines.

## 2.2 Semantic Retrieval and $k$NN Adaptation
To mitigate hallucination and ground model behavior in factual intelligence, the AI community has widely adopted Retrieval-Augmented Generation (RAG) paradigms. The foundation of modern retrieval-augmented decoding traces back to the $k$NN-LM framework originally proposed by Khandelwal et al. (2020), which demonstrated that interpolating parametric predictions with non-parametric FAISS datastore retrievals drastically reduces testing perplexity.

In 2024 and 2025, retrieval mechanisms have become ubiquitous in Software Engineering, heavily utilized in applications like RepoCoder and various code completion frameworks to inject repository-level context at inference time. However, the vast majority of recent related work focuses exclusively on using retrieval to augment *generative* tasks. Our CI-Gated $k$NN framework uniquely adapts this paradigm for *classification* safety. Rather than executing computationally heavy retrieval for every query, our architecture implements an adaptive Predictive Entropy sigmoid gate. This formulation explicitly reserves $k$NN interpolation as a secure fallback mechanism exclusively for when the base parametric model detects high internal uncertainty.

## 2.3 Conformal Prediction for Software Engineering
As the necessity for AI safety has escalated, Conformal Prediction (CP)—a framework for distribution-free uncertainty quantification (Angelopoulos & Bates, 2021)—has emerged as a foundational solution for modern machine learning reliability. Conformal prediction transforms deterministic, point-estimate models into mathematically guaranteed prediction sets that contain the true label with a high, user-defined probability (e.g., $\ge 90\%$).

In 2024 and 2025, CP has seen explosive adoption across generative and natural language tasks, evidenced by major frameworks like Conformal Language Modeling (CLM) (ICLR 2024), Conformal Structured Prediction (ICLR 2025), and Generative Prediction Sets (GPS) for Code Generation (2025). These contemporary works establish stopping rules and prediction sets to guarantee functional code correctness. 

However, applying conformal control specifically to discriminative code-classification tasks (defect and vulnerability detection) remains under-explored; our own web survey found no prior work combining predictive-entropy gating, $k$NN retrieval, and split conformal prediction for these tasks. In security and quality-gating contexts, single-class argmax predictions are unsafe. Our CI-Gated $k$NN applies Split Conformal Prediction to bound the combined $p_{final}$ distribution, providing distribution-free coverage that holds under semantic-preserving perturbations. We scope this claim carefully: the framework is effective on multi-class defect prediction, whereas binary vulnerability detection is reported as a negative result (§4.2), which we attribute to the near-uniform entropy of two-class softmax.

## 2.4 Program Adaptation for Code Models (CodeImprove)
The closest prior work is CodeImprove (Rathnasuriya et al., ICSE 2025), which restores degraded code-model performance under distribution shift *without retraining* by (i) detecting out-of-scope inputs with a validity score (reported AUC 0.924) and (ii) transforming such inputs back in-scope via semantic-preserving program transformations. CodeImprove operates on the **input** side and targets point accuracy (up to +8.78%). Our work is **complementary**: it operates on the **output** side, leaving the input untouched, and contributes calibration and a distribution-free coverage guarantee that CodeImprove does not address. We do not claim to outperform CodeImprove on detection or accuracy; rather, the two mechanisms are orthogonal and could in principle be composed.
